## 📖 프로젝트 개요
**제주국제학교에 재학 중인 지인의 요청으로, 수강 정정 커뮤니케이션을 효율화하기 위해
기획·개발·운영 전 과정을 주도한 웹 기반 메일 발송 및 관리 시스템입니다.**
<br>
IT 직군을 희망하는 지인과 프로젝트 전반의 과정을 공유하며 진행하였고,
실제 운영 환경에서 안정적으로 활용될 수 있도록 구현하여
서비스 운영까지 성공적으로 마쳤습니다.

<br><br>

## 🖥️ 개발 환경
- **Back-end**: Java
- **Front-end**: HTML/CSS, Bootstrap, JavaScript, jQuery
- **View**: JSP
- **Framework**: Spring Boot, MyBatis
- **DB**: PostgreSQL
- **Web Server**: Nginx
- **WAS**: Tomcat
- **Infra**: Naver Cloud Platform(NCP)
- **Domain / SSL**: 도메인 연결 및 Let’s Encrypt 기반 HTTPS 구성
- **Dev Tool**: IntelliJ IDEA (Gemini 활용)
- **Version Control**: GitHub

<br><br>



<br><br>

## 💡 전체 구현 기능
* 🔐 Google OAuth 2.0 기반 로그인 / 사용자 인증
* 👤 학생 사용자 관리 (구글 계정 정보 연동)
* 📚 강의 목록 조회 및 학기별 강의 정보 관리
* ✏️ 수강 신청 및 수강 정정 요청 기능
* 📩 수강 정정 요청 기반 자동 메일 발송 (Gmail 연동)
* 📨 메일 발송 이력 조회 및 관리
* 📊 수강 정정 변경 이력 관리
* 🔍 사용자/관리자 권한 분리 및 접근 제어
* 🌐 도메인 기반 웹 서비스 운영 (HTTPS 적용)
* 🛠️ 관리자 모드
  - 강의 정원(최대 수강 인원) 수정
  - 수강 인원 변경 이력 관리

**[메일발송]**
* 학생이 최종 수강 정정 내용을 확인한 후, Send 버튼을 클릭하면 메일 발송 팝업이 표시
* 팝업 화면에는 학생 정보(이름, 이메일, 학년) 와 변경 전·후 수강 과목 목록이 요약되어 출력
* 메일 제목과 본문은 기본 템플릿으로 자동 생성되며, 필요 시 내용 수정 가능
* 수강 정정 요청 메일을 담당 교사 및 관리자에게 발송
* 메일 발송 완료 후, 발송 결과 및 이력은 시스템에 저장되어 추후 조회가 가능

## 📚 프로젝트 후기

이번 프로젝트는 단순히 기능을 구현하는 데 그치지 않고,
기획 → 설계 → 개발 → 배포 → 운영까지 전 과정을 직접 경험한 프로젝트였습니다.

실제 교육 환경에서 발생하는 수강 정정 커뮤니케이션의 불편함을 개선하기 위해 시작된 프로젝트로,
요구사항이 처음부터 명확하지 않은 상태에서 사용자와 지속적으로 소통하며 기능을 구체화해 나갔습니다.
이 과정에서 “개발자는 코드를 작성하는 사람을 넘어 문제를 정의하고 해결하는 역할을 한다”는 점을 깊이 체감할 수 있었습니다.

개발 과정에서는 Google OAuth 로그인, Gmail API를 활용한 메일 발송,
수강 과목 변경 로직 및 관리자 기능 구현 등 다양한 기술 요소를 실제 서비스 흐름에 맞게 적용하였고,
특히 운영 중 발생하는 데이터 중복, 정합성 문제, 권한 분리와 같은 이슈를 직접 해결하며
단순 구현이 아닌 운영 관점의 개발 경험을 쌓을 수 있었습니다.

또한 클라우드 서버 환경에서 직접 서비스를 운영하며
Nginx 리버스 프록시 구성, HTTPS(SSL 인증서) 적용, 서버 설정 및 로그 확인 등
백엔드 개발자에게 필요한 인프라 전반의 흐름을 이해하게 되었습니다.
“코드는 로컬에서 끝나는 것이 아니라, 서버에서 안정적으로 돌아가야 한다”는 점을 몸소 경험한 프로젝트였습니다.

무엇보다 의미 있었던 점은,
IT 직무를 희망하는 지인과 프로젝트를 함께 진행하며 설계 의도와 구현 과정을 공유하고
실제 사용 가능한 서비스로 완성해냈다는 점입니다.
이 경험을 통해 기술을 설명하고 함께 성장하는 개발자의 역할에 대해 다시 한 번 생각하게 되었습니다.
